# PORT vs PASV
PORT:positive主动模式 服务器 连 客户端的数据端口
PASV:passive 被动模式 客户端 连 服务器的数据端口

# 为什么要有这两种模式?
## FTP天生两条独立TCP连接：
控制连接（一直在线，发命令）
数据连接（临时建立，传文件 / 目录列表）
## 问题来了：
数据连接，到底该谁主动发起TCP连接？
就因为「谁主动连谁」这件事，分出了PORT主动/PASV被动两种模式

# 什么时候用哪个?
只要是内网、校园网、家用网、公司网 → 必用被动模式PASV
只有双方都是公网独立IP、无防火墙、无NAT → 才用主动模式PORT

# 主动模式和被动模式的具体工作流程是怎样的？
## 主动模式
客户连服务控制口 → 客户给自己开数据口 → 发 PORT 告诉服务 → 服务主动连客户数据口 → 传数据
## 被动模式
客户连服务控制口 → 客户发 PASV → 服务给自己开数据口 → 发 227 告诉客户 → 客户主动连服务数据口 → 传数据

# 代码实现
PASV:
ftp_client_pasv.cpp 功能：连接 2100 → 发 PASV → 解析 227 → 连接服务端数据端口 → 接收数据
ftp_server_pasv.cpp 功能：监听 2100 端口 → 接收 PASV → 开随机数据端口 → 等待客户端连接
PORT:
ftp_client_port.cpp 功能：连接 2100 → 开本地数据端口 → 发 PORT → 等待服务端连接 → 接收数据
ftp_server_port.cpp 功能：监听 2100端口 → 接收 PORT → 解析客户端数据端口→ 主动连接客户端 → 发送数据
